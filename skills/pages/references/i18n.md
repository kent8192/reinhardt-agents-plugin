# Reactive Pages I18n Reference

This reference applies to the 0.4.x Pages i18n surface introduced by the
reactive translation runtime. For 0.1.x through 0.3.x, use the server-function
translation boundary described in `head-form-macros.md` instead.

## Enable Pages I18n

Reactive Pages translations are feature-gated. Add `i18n` alongside `pages` on
the facade dependency; enabling `i18n` by itself does not activate the Pages
translation module.

```toml
[dependencies]
reinhardt = { version = "...", features = ["pages", "i18n"] }
reinhardt-i18n = "..."
```

Use versions compatible with the application release line. Depend directly on
`reinhardt-i18n` when application code constructs `MessageCatalog` or
`TranslationContext`; its catalog types are needed by both native SSR setup and
shared Pages code.

## Render Catalog-backed UI Text

Create an `I18nContext` from the application catalogs and provide it for the
page render scope. The `t!` macro returns lazy `TranslatedText`, so the message
is resolved at render time and follows locale changes without threading a
`Resource` through every component.

```rust
use reinhardt::pages::{prelude::*, t};
use reinhardt_i18n::{MessageCatalog, TranslationContext};

let mut translations = TranslationContext::new("ja", "en-US");
let mut japanese = MessageCatalog::new("ja");
japanese.add_translation("Workspace", "ワークスペース");
japanese.add_translation("Welcome, {name}", "ようこそ、{name}さん");
translations
    .add_catalog("ja", japanese)
    .expect("catalog locale must be valid");

let i18n_context = I18nContext::new(translations);
let _i18n = provide_i18n_context(i18n_context.clone());

let user_name = "Ada".to_string();
let page = page!(|user_name: String| {
    main {
        h1 { { t!("Workspace") } }
        p { { t!("Welcome, {name}", name = user_name) } }
    }
})(user_name);
```

`t!` accepts a literal message ID plus optional named interpolation arguments.
For plural or contextual messages, use the lazy helpers directly:

```rust
let item_count = 3_usize;

page!(|item_count: usize| {
    p { { tn("{count} item", "{count} items", item_count).arg("count", item_count) } }
    p { { tp("button", "Open") } }
    p { { tnp("files", "{count} file", "{count} files", item_count).arg("count", item_count) } }
})(item_count);
```

Keep message IDs literal and stable for catalog extraction. Do not use a
user-controlled value as a translation key.

## Change Locale Reactively

Use the validated context API for locale writes. Existing `t!` nodes read the
locale reactively and update when the page renders again.

```rust
let active_locale = i18n_context.locale();
i18n_context
    .set_locale("fr")
    .expect("supported locale must be valid");
```

Do not access or mutate a locale signal directly: the public writable
`locale_signal()` accessor was removed. `set_locale()` validates locale tags and
preserves the page-scoped context contract.

## SSR and Hydration

Pass the same context to `SsrOptions` for SSR:

```rust
let mut renderer = SsrRenderer::with_options(
    SsrOptions::new().i18n_context(i18n_context),
);
```

This renders translated text synchronously, sets `<html lang>` from the active
locale, and serializes the resolved catalogs under `pages.i18n`. Standard Pages
hydration restores that metadata before the first client render. Do not add a
separate client catalog request or duplicate fallback string for an
SSR-resolved catalog.

This contract only covers catalogs already available to the context. If copy is
derived from server-only policy, request data, authorization, or a remote source,
use a focused `#[server_fn]` and `use_resource` with a stable fallback instead.

## Review and Test Checklist

- Verify the facade enables both `pages` and `i18n`.
- Verify simple translated UI uses `t!`; use `tn`, `tp`, or `tnp` for plural or
  contextual forms rather than inventing unsupported `t!` syntax.
- Verify locale changes call `set_locale()` and translated output changes.
- For SSR, verify translated HTML, `pages.i18n`, and `<html lang>` agree.
- Verify the first hydrated render reuses the SSR catalog without a fetch or
  untranslated-content flash.
- Mark tests that mutate shared i18n state with `#[serial(i18n)]`.
